[−][src]Module greenhorn::dom

This module exposes the DomEvent data type which is used to communicate DOM events from the frontend to the backend.

DomEvents are the principal data type for frontend to backend communication. When subscribing to a event on the DOM a handler function is registered in the application runtime, which maps a DomEvent to a the message type of the running [../trait.App.html].

When the subscribed event is triggered on the frontend, a DomEvent is created, mapped with the handler function and subsequently passed into the update() cycle of the application.
